# mlp
我运行了一个简单的MLP代码，并通过不断修改代码中的参数来观察模型训练过程中 loss 的变化情况。在实验过程中，我重点尝试了修改学习率、epoch 以及网络结构等内容。当我将学习率调得过大时，loss 会出现明显震荡，甚至直接变成非常大的数或者nan，这说明参数更新幅度过大，模型无法稳定收敛；而当学习率调得过小时loss虽然能够下降，但下降速度非常慢，模型训练效率较低。理解了学习率实际上控制的是参数每次更新的步长，对模型训练稳定性影响极大。我还尝试修改epoch数量。当epoch较少时，模型还没有充分学习数据规律，loss下降并不明显；而增加epoch后，可以看到loss持续减小，预测结果也逐渐接近真实值。我理解到epoch本质上表示模型对整个数据集重复学习的次数，适当增加训练轮数能够提升模型效果，但过多时也可能带来过拟合问题。我还尝试修改网络结构，例如增加隐藏层和 ReLU 激活函数，发现，仅使用单层 Linear 时模型只能学习简单的线性关系，而加入ReLU后，网络具备了学习非线性规律的能力，loss下降效果也更明显。激活函数在深度学习中的重要作用，让神经网络不再只是简单的线性叠加。在代码层面，我还通过注释 backward()、optimizer.step()、zero_grad() 等语句来观察程序运行结果的变化。注释 backward() 后，模型无法进行反向传播，loss 基本不会下降；注释 optimizer.step() 后，虽然能够计算梯度，但参数不会更新，模型同样无法学习；而去掉 zero_grad() 后，梯度会不断累积，导致训练结果出现异常。
